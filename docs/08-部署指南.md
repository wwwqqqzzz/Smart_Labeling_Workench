## 开发环境部署

### 使用Docker Compose（推荐）

#### 1. 准备工作

```bash
# 克隆项目
git clone <repository-url>
cd Smart-Labeling-Workbench

# 复制环境变量文件
cp backend/.env.example backend/.env
cp frontend/.env.local.example frontend/.env.local
```

#### 2. 配置环境变量

编辑 `backend/.env`:
```bash
# API Keys
GLM_API_KEY=your_glm_api_key_here
OPENAI_API_KEY=your_openai_api_key_here

# 数据库
DATABASE_URL=postgresql://user:password@postgres:5432/smartlabeling

# CORS
BACKEND_CORS_ORIGINS=["http://localhost:3000","http://localhost:3002"]
```

编辑 `frontend/.env.local`:
```bash
NEXT_PUBLIC_API_URL=http://localhost:8000
```

#### 3. 启动服务

```bash
# 构建并启动所有服务
docker-compose up -d

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

#### 4. 初始化数据库

```bash
# 进入后端容器
docker-compose exec backend bash

# 初始化数据库
python scripts/init_db.py

# 生成测试数据（可选）
python scripts/generate_test_data.py 50

# 退出容器
exit
```

#### 5. 访问服务

- 前端: http://localhost:3002
- 后端API: http://localhost:8000
- API文档: http://localhost:8000/docs

## 生产环境部署

### 方案1: Docker Compose（小型部署）

#### 1. 使用生产配置

```bash
# 使用生产配置文件
docker-compose -f docker-compose.prod.yml up -d
```

#### 2. 配置SSL证书

```bash
# 使用Let's Encrypt获取免费SSL证书
certbot certonly --standalone -d your-domain.com

# 证书路径
/etc/letsencrypt/live/your-domain.com/fullchain.pem
/etc/letsencrypt/live/your-domain.com/privkey.pem
```

#### 3. 更新Nginx配置

```nginx
server {
    listen 443 ssl http2;
    server_name your-domain.com;

    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;

    # 前端
    location / {
        proxy_pass http://frontend:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }

    # 后端API
    location /api/ {
        proxy_pass http://backend:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # WebSocket支持（如果需要）
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
    }
}
```

## Docker优化

### 1. 多阶段构建

#### 后端 Dockerfile

```dockerfile
# 构建阶段
FROM python:3.11-slim as builder

WORKDIR /app

# 安装构建依赖
RUN apt-get update && apt-get install -y \
    gcc \
    postgresql-client \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir --user -r requirements.txt

# 运行阶段
FROM python:3.11-slim

WORKDIR /app

# 复制已安装的包
COPY --from=builder /root/.local /root/.local

# 复制应用代码
COPY . .

# 确保Python能找到已安装的包
ENV PATH=/root/.local/bin:$PATH

# 暴露端口
EXPOSE 8000

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD curl -f http://localhost:8000/health || exit 1

# 启动命令
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8000"]
```

#### 前端 Dockerfile

```dockerfile
# 构建阶段
FROM node:18-alpine as builder

WORKDIR /app

# 复制package文件
COPY package*.json ./

# 安装依赖
RUN npm ci

# 复制源代码
COPY . .

# 构建应用
RUN npm run build

# 运行阶段
FROM node:18-alpine

WORKDIR /app

# 复制构建产物
COPY --from=builder /app/.next ./.next
COPY --from=builder /app/node_modules ./node_modules
COPY --from=builder /app/package.json ./package.json
COPY --from=builder /app/public ./public

# 暴露端口
EXPOSE 3000

# 健康检查
HEALTHCHECK --interval=30s --timeout=3s --start-period=5s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:3000 || exit 1

# 启动命令
CMD ["npm", "start"]
```

### 3. Docker Compose优化

```yaml
version: '3.8'

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
      cache_from:
        - smart-labeling-backend:latest
    image: smart-labeling-backend:latest
    restart: unless-stopped

      - DATABASE_URL=postgresql://user:pass@postgres:5432/db

      postgres:
        condition: service_healthy

        reservations:
          cpus: '0.25'

    build:

      dockerfile: Dockerfile
      cache_from:
        - smart-labeling-frontend:latest
    image: smart-labeling-frontend:latest
    restart: unless-stopped
    depends_on:
      - backend
    networks:
      - app-network
    deploy:
      resources:
        limits:
          cpus: '0.5'
          memory: 512M
        reservations:
          cpus: '0.25'
          memory: 256M

  postgres:
    image: postgres:15-alpine
    restart: unless-stopped
    environment:
      - POSTGRES_DB=smartlabeling
      - POSTGRES_USER=admin
      - POSTGRES_PASSWORD=password
    volumes:
      - postgres_data:/var/lib/postgresql/data
    networks:
      - app-network
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U admin"]
      interval: 10s
      timeout: 5s
      retries: 5

volumes:
  backend_data:
  postgres_data:

networks:
  app-network:
    driver: bridge
```

## 备份策略

### 1. 数据库备份

#### 自动备份脚本

```bash
#!/bin/bash
# backup.sh

DATE=$(date +%Y%m%d_%H%M%S)
BACKUP_DIR="/backups"
DB_NAME="smartlabeling"

# PostgreSQL备份
docker exec postgres pg_dump -U admin $DB_NAME | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# 保留最近7天的备份
find $BACKUP_DIR -name "db_*.sql.gz" -mtime +7 -delete

echo "Backup completed: db_$DATE.sql.gz"
```

#### 定时任务

```bash
# 添加到crontab
crontab -e

# 每天凌晨2点备份
0 2 * * * /path/to/backup.sh >> /var/log/backup.log 2>&1
```

### 2. 向量数据库备份

```bash
# Chroma备份
docker exec backend tar czf /tmp/chroma_backup_$DATE.tar.gz /app/data/chroma
docker cp backend:/tmp/chroma_backup_$DATE.tar.gz ./backups/
```

### 3. 恢复流程

```bash
# 恢复PostgreSQL
gunzip < db_20260113_020000.sql.gz | docker exec -i postgres psql -U admin smartlabeling

# 恢复Chroma
docker cp ./backups/chroma_backup.tar.gz backend:/tmp/
docker exec backend tar xzf /tmp/chroma_backup.tar.gz -C /
```

## 故障排查

### 常见问题

```bash
# 查看容器日志
docker-compose logs backend
docker-compose logs frontend

# 检查容器状态
docker-compose ps

docker-compose exec backend bash
```

#### 2. 数据库连接失败

```bash
# 检查数据库状态
docker-compose exec postgres pg_isready

# 查看数据库日志
docker-compose logs postgres

# 测试连接
docker-compose exec backend python -c "from app.database import engine; print('Connected')"
```

```bash
# 查看后端详细日志
docker-compose exec backend tail -f /var/log/app.log

docker-compose exec backend env | grep DATABASE_URL
```

```bash
docker-compose exec frontend ls -la .next

docker-compose exec frontend sh -c 'curl http://backend:8000/health'
```
